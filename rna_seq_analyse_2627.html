Introductie

In dit project wordt onderzocht welke genen verschillend tot expressie komen tussen hersenweefsel en leverweefsel. De onderzoeksvraag luidt: welke verschillen in genexpressie zijn aanwezig tussen hersen- en leverweefsel en welke biologische processen hangen hiermee samen?

Om deze vraag te beantwoorden wordt RNA-seq data geanalyseerd. De reads worden uitgelijnd tegen een referentiegenoom en samengevat in een counttabel, waarna een differentiële genexpressieanalyse wordt uitgevoerd. Vervolgens wordt een GO-term enrichment analyse toegepast om inzicht te krijgen in de biologische functies en processen die samenhangen met de gevonden expressieverschillen.


Alignment

Voor de alignment zijn de FASTQ-bestanden van lever- en hersenweefsel uitgelijnd tegen het humane referentiegenoom hg38 met de functie align() uit het Rsubread pakket. Hiervoor is een vooraf gegenereerde hg38-index gebruikt. De zes FASTQ-bestanden (drie lever en drie brein) zijn meegegeven via readfile1, zodat elke sample apart wordt uitgelijnd.

Omdat het om RNA-seq data gaat, is type = 0 ingesteld zodat het juiste mappingprotocol voor RNA wordt toegepast. De inputbestanden zijn gzipped FASTQ-bestanden (input_format = “gzFASTQ”) en de resultaten zijn opgeslagen als BAM-bestanden (output_format = “BAM”).

Met unique = TRUE zijn alleen reads meegenomen die op één duidelijke plek in het genoom mappen. Reads die op meerdere locaties kunnen mappen zijn uitgesloten, zodat elke read eenduidig wordt toegewezen. De alignment statistieken zijn opgeslagen in het object alignment_statistics voor verdere analyse.

Er is gekozen voor uniquely mapped reads, omdat expressieverschillen tussen verschillende weefsels worden vergeleken en hiervoor een precieze koppeling van reads aan genen nodig isDit heeft als resultaat dat betrouwbaarheid van de genkwantificatie wordt vergroot, waardoor verschillen in expressie tussen weefsels nauwkeuriger geïnterpreteerd kunnen worden.

Count tabel en data inspectie

Voor het genereren van de count table zijn de zes BAM-bestanden van lever- en breinsamples geanalyseerd met de functie featureCounts() uit het Rsubread pakket, waarbij gebruik is gemaakt van de ingebouwde hg38 RefSeq annotatie. Er is gekozen voor useMetaFeatures = TRUE zodat reads op genniveau worden samengevoegd, wat aansluit bij het doel om genexpressie tussen weefsels te vergelijken. De parameter strandSpecific = 0 is ingesteld omdat de libraries niet strand-specifiek zijn voorbereid, waardoor reads van beide DNA-strengen mogen worden meegeteld. Daarnaast is isPairedEnd = FALSE gebruikt omdat de data uit single-end sequencing bestaat.

De verkregen countmatrix is vervolgens omgezet naar een DESeqDataSet, waarbij in de metadata per sample is vastgelegd of deze afkomstig is van lever- of breinweefsel. In het design is ~ weefsel opgenomen, zodat specifiek het verschil tussen deze twee weefsels wordt getest.

Voor de data-inspectie is gekozen voor een variance stabilizing transformatie (vst) in plaats van een eenvoudige log-transformatie. Een log-transformatie corrigeert wel voor grote verschillen in schaal, maar houdt onvoldoende rekening met de mean-variance relatie die kenmerkend is voor RNA-seq data. De vst-methode stabiliseert de variantie over het hele bereik van expressiewaarden en maakt lage en hoge expressieniveaus beter vergelijkbaar. Hierdoor is de data geschikter voor multivariate analyses zoals PCA en correlatieanalyse. Bij het uitvoeren van de vst-transformatie is gekozen voor blind = FALSE, zodat de transformatie rekening houdt met het experimentele design (~ weefsel). Hierdoor blijven de biologische verschillen tussen lever- en brein-samples behouden, wat van belang is voor het correct interpreteren van PCA en heatmaps. Ter vergelijking zou blind = TRUE de transformatie uitvoeren zonder kennis van de groepsindeling, waardoor verschillen tussen condities worden afgevlakt. Omdat het doel van deze analyse juist is om weefselverschillen zichtbaar te maken, is blind = FALSE de juiste keuze.

De PCA is uitgevoerd om te beoordelen of samples van hetzelfde weefsel bij elkaar clusteren en of er een duidelijke scheiding zichtbaar is tussen lever en brein. Daarnaast is een correlatiematrix berekend met de functie cor() op de vst-getransformeerde waarden en gevisualiseerd met pheatmap(). Hiermee kan worden gecontroleerd of replicaten binnen hetzelfde weefsel sterk correleren en of de globale expressiepatronen consistent zijn.

Genereren van count table

Omzetten van de count tabel naar een DESeqDataSet

# Countmatrix uit list halen
countDatamatrix <- read_counts$counts

# Metadata maken/definieren
metadata_coldata <- data.frame(
  row.names = colnames(countDatamatrix),
  weefsel = c("Lever", "Lever", "Lever", "Brein", "Brein", "Brein")
)

dds <- DESeqDataSetFromMatrix(
  countData = countDatamatrix, # De count tabel in de vorm van een matrix
  colData = metadata_coldata, # De metadata van de count tabel in de vorm van een dataframe, zodat voor elk sample aangegeven wordt uit welk weefsel deze afkomstig is.
  design = ~ weefsel # Een formule die aangeeft welk verschil er getest moet worden, in dit geval is dat verschil: type weefsel.
)


Data inspectie mbv correlatieheatmap en PCA

# Data inspectie dmv PCA

# De variance stabilizing transformatie corrigeert voor verschillen in sequencingdiepte en zorgt ervoor dat de spreiding van lage en hoge expressiewaarden beter vergelijkbaar wordt.
vsd <- vst(dds, blind = FALSE) # blind = FALSE, omdat DESeq2 dan rekening houdt met het design (~ weefsel) en de biologische verschillen tussen lever en brein zichtbaar blijven


# De getransformeerde expressiewaarden worden uit het DESeqDataSet gehaald in de vorm van een matrix, zodat deze gebruikt kunnen worden voor de PCA-analyse.
vsd_matrix <- assay(vsd)

# De principal component analyse berekent nieuwe variabelen (PC's) die de grootste verschillen tussen de samples samenvatten.
pca <- prcomp(t(vsd_matrix))

# De PC1- en PC2-waarden van elk sample worden samengevoegd met het bijbehorende type weefsel in één dataframe.
pca_plotting <- data.frame(
  PC1 = pca$x[, 1],
  PC2 = pca$x[, 2],
  weefsel = metadata_coldata$weefsel
)

# De namen van de samples worden toegevoegd aan het dataframe.
pca_plotting$sample <- rownames(pca$x)

# Het percentage verklaarde variantie van PC1 en PC2 wordt berekend om te laten zien hoeveel van de totale spreiding wordt verklaard.
PC1_var <- round(100 * summary(pca)$importance[2, 1], 1)
PC2_var <- round(100 * summary(pca)$importance[2, 2], 1)

# De PCA-plot toont PC1 op de x-as en PC2 op de y-as, waarbij de kleuren het type weefsel aangeven.
ggplot(pca_plotting, aes(x = PC1, y = PC2)) +
  geom_point(aes(color = weefsel), size = 3) +
  scale_color_manual(values = c(
    "Brein" = "#007A4D",
    "Lever" = "#F0E442"
  )) +
  ggtitle("PCA van Lever- en Brein-samples") +
  xlab(paste0("PC1 (", PC1_var, "%)")) +
  ylab(paste0("PC2 (", PC2_var, "%)")) +
  theme_bw() +
  theme(
    plot.title = element_text(hjust = 0.5, face = "bold"),
    text = element_text(size = 14)
  )

Principal Component Analysis (PCA) van RNA-seq data.

Figuur 2 toont het PCA-plot die een duidelijke scheiding laat zien tussen lever- en breinsamples. De leversamples (geel) liggen links in de grafiek en clusteren dicht bij elkaar, terwijl de breinsamples (groen) rechts liggen en eveneens een aparte cluster vormen (met een enkel punt die afwijkt van de twee andere breinsamples). PC1 verklaart het grootste deel van de variantie en scheidt de twee weefsels duidelijk van elkaar.


# Data inspectie dmv correlatiematrix en heatmap

# De variance stabilizing transformatie corrigeert voor verschillen in sequencingdiepte en maakt de expressiewaarden beter vergelijkbaar.
vsd <- vst(dds, blind = FALSE) # blind = FALSE, omdat DESeq2 dan rekening houdt met het design (~ weefsel) en de biologische verschillen tussen lever en brein zichtbaar blijven. 

# De getransformeerde expressiewaarden worden als matrix opgehaald uit het DESeqDataSet object.
vsd_matrix <- assay(vsd)

# De correlatie tussen alle samples wordt berekend op basis van de expressiewaarden.
cor_matrix <- cor(vsd_matrix)

# Annotatie dataframe maken waarin per sample het type weefsel staat aangegeven.
annotation_col <- data.frame(
  Weefsel = metadata_coldata$weefsel
)
rownames(annotation_col) <- colnames(vsd_matrix)

# Kleuren definiëren voor de weefsels.
annotation_colors <- list(
  Weefsel = c(
    "Brein" = "#007A4D",
    "Lever" = "#F0E442"
  )
)

# De correlatieheatmap visualiseert de overeenkomst tussen samples, waarbij de annotatie het type weefsel aangeeft.
pheatmap(
  cor_matrix,
  annotation_col = annotation_col,
  annotation_colors = annotation_colors,
  main = "Correlatie tussen Lever- en Brein-samples",
  fontsize_row = 10,
  fontsize_col = 10,
  border_color = NA
)

Correlatieheatmap van de vst-getransformeerde expressiewaarden.

De correlatieheatmap in figuur 3 toont dat de drie leversamples onderling een hoge correlatie hebben en samen clusteren, terwijl de drie breinsamples eveneens sterk met elkaar correleren en een aparte cluster vormen. De correlatie tussen lever- en breinsamples is duidelijk lager. De dendrogrammen laten zien dat de samples zich groeperen op basis van weefseltype.


Conclusie van de correlatieheatmap en PCA

Beide figuren laten zien dat de samples clusteren op basis van weefseltype. Replicaten binnen hetzelfde weefsel vertonen sterke overeenkomst, terwijl lever- en breinweefsel duidelijk van elkaar verschillen in genexpressie. Dit wijst op consistente metingen en duidelijke biologische verschillen tussen de twee weefsels.
